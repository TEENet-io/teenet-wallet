# 使用指南

## 什么是 TEENet Wallet？

TEENet Wallet 是专为 OpenClaw 设计的安全多链加密货币钱包。OpenClaw 是 Telegram 上的 AI 助手，你在聊天中告诉它你想做什么 —— 转账、查余额、交互 DeFi —— 它通过 TEENet Wallet 帮你完成。你不需要研究合约参数或手动填写交易表单，和 OpenClaw 聊天就够了。

你的私钥由分布式 TEE 安全硬件保护，永远不会完整存在于任何一台设备上。大额交易和敏感操作需要你用 Passkey（指纹或面容）亲自确认，OpenClaw 无法绕过这道关卡。小额日常操作则由 OpenClaw 即时完成，不打扰你。

---

## 工作原理

你在 Telegram 上和 OpenClaw 对话，就像和一个懂加密货币的朋友聊天。你说"转 0.1 ETH 到某个地址"，OpenClaw 就用你的钱包帮你完成。

OpenClaw 可以帮你发送加密货币、查询余额、交互 DeFi 协议、管理代币 —— 所有操作都通过 Telegram 聊天完成。

**小额操作，OpenClaw 即时执行。** 低于你设定阈值的转账，OpenClaw 直接完成，几秒钟内你就能在聊天中看到结果。

**大额操作，你来拍板。** 超过阈值的交易会暂停，OpenClaw 会在 Telegram 里发给你一个审批链接。点开链接，用指纹或面容确认，交易才会执行。

**就像有一个懂加密货币的助手，消费额度由你控制。** 你给助手设定一个限额，日常采购它可以自己决定；但要花大钱，它必须先问你。你随时可以查看每一笔支出记录，也可以随时调整限额或收回授权。

## 开始使用

### 第一步：创建账户

1. 在浏览器中打开 https://wallet.example.com ，推荐使用 Chrome。

2. 点击顶部的 **Register**（注册）按钮。

3. 输入一个显示名称，比如"alice"或你的昵称。

4. 点击 **Register Device**（注册设备）。浏览器会弹出提示，让你设置 Passkey —— 根据设备不同，可能是指纹识别、面容识别、硬件安全密钥或设备 PIN 码。

5. 按提示完成验证，账户就创建好了。

**什么是 Passkey？** Passkey 是一种比密码更安全的登录方式，利用你设备上的生物识别（指纹或面容）来验证身份。你不需要记住任何密码，凭证存储在你的设备上，无法被钓鱼或泄露。在 TEENet Wallet 中，Passkey 是你审批交易和确认敏感操作的唯一方式 —— 它就是你作为钱包主人的身份凭证。

### 第二步：生成 API 密钥

API 密钥是 OpenClaw 访问你钱包的凭证。没有密钥，OpenClaw 无法执行任何操作。

1. 登录后，切换到 **Account**（账户）标签页。

2. 在 API 密钥区域，输入一个标签名（可选），比如"my-openclaw"。

3. 点击 **Generate API Key**（生成 API 密钥），通过 Passkey 验证。

4. 你会看到生成的密钥（以 **ocw_** 开头）。请立即复制并安全保存 —— 它只显示一次，之后无法再查看。

### 第三步：在 OpenClaw 上安装钱包技能

1. 打开 Telegram，找到 OpenClaw 的对话。

2. 发送这条消息：

**"安装这个 skill: https://github.com/TEENet-io/teenet-wallet/blob/master/skill/tee-wallet/SKILL.md"**

3. OpenClaw 会识别这个技能，并要求你提供两个设置：

   - **TEE_WALLET_API_URL** —— 输入钱包地址：https://wallet.example.com
   - **TEE_WALLET_API_KEY** —— 粘贴你在第二步复制的 API 密钥

4. 设置完成后，OpenClaw 就具备了操作你钱包的能力。

### 第四步：创建钱包

现在可以让 OpenClaw 帮你创建钱包了。在 Telegram 里告诉它：

- **"创建一个以太坊钱包"**
- **"创建一个 Solana 钱包"**
- **"帮我创建一个叫'日常开销'的以太坊钱包"**

OpenClaw 会创建钱包并显示地址。以太坊钱包大约需要 1 分钟（多个安全节点在协作生成密钥碎片），Solana 钱包通常几秒内完成。

拿到地址后，向其充值即可开始使用。

你也可以在 Web 界面（https://wallet.example.com）上手动创建钱包，效果完全一样。

### 第五步：设置消费限额

消费限额决定了 OpenClaw 的自主权边界 —— 多大金额以内它可以自己完成，超过多少需要你亲自确认。

**通过 Web 界面设置：**

1. 登录 https://wallet.example.com ，在钱包列表中展开目标钱包。
2. 切换到 **Policy**（策略）标签页。
3. 设置 **USD 阈值**，比如 100 美元。OpenClaw 发起的交易低于这个金额可以直接执行，超过则需要你审批。
4. 可选：设置 **日限额**，比如 5000 美元。当天交易总额达到上限后，OpenClaw 无法再发送任何交易。
5. 点击 **Save** 保存。

**通过 Telegram 设置：** 你也可以直接告诉 OpenClaw：**"设置审批阈值为 100 美元"**。

建议从较低的阈值开始，观察一段时间后再根据需要调整。

## 和 OpenClaw 对话

这是你使用 TEENet Wallet 的主要方式。以下是一些具体的 Telegram 对话示例，展示 OpenClaw 能帮你做什么。

### 钱包管理

- **"创建一个叫'交易用'的以太坊钱包"** —— OpenClaw 会创建钱包并返回地址
- **"显示我的钱包"** —— 列出你所有钱包的地址、链和名称
- **"我的余额是多少？"** —— 查询当前钱包的链上余额
- **"显示所有钱包余额"** —— 一次性查看所有钱包的资产情况

### 发送加密货币

- **"转 0.1 ETH 到 0xABC...123"** —— 发送以太坊原生代币
- **"发送 50 USDC 到 0xDEF...456"** —— 发送 ERC-20 代币
- **"转 1 SOL 到 ABC...XYZ"** —— 发送 Solana 原生代币

如果转账金额低于你设定的阈值，OpenClaw 会即时完成并告诉你交易哈希。如果超过阈值，OpenClaw 会在聊天中发给你一个审批链接，等你用指纹确认后再执行。

### DeFi 和合约交互

- **"在 Uniswap 上把 0.5 ETH 换成 USDC"** —— 执行去中心化交易
- **"授权 Uniswap 路由器使用我的 USDC"** —— 代币授权操作
- **"查询 USDC 合约上我的余额"** —— 读取合约数据

合约交互需要目标合约在白名单中。如果合约不在白名单里，OpenClaw 会自动发起添加提案，你在审批中确认即可。

### 白名单管理

- **"把 USDC 合约加到白名单"** —— OpenClaw 发起添加提案
- **"显示白名单里的合约"** —— 查看已授权的合约列表

注意：添加白名单需要你在 Web 界面或通过审批链接用 Passkey 确认。这个流程确保每一个新合约都经过你的明确授权，OpenClaw 无法自行扩展权限范围。

你可以在 Web 界面上删除白名单中的合约，移除后 OpenClaw 将无法再与该合约交互。

### 策略和安全

- **"设置审批阈值为 200 美元"** —— 调整消费限额
- **"显示我的审批策略"** —— 查看当前阈值和日限额设置
- **"显示待审批的交易"** —— 查看等待你确认的交易

## 审批交易

当 OpenClaw 发起超过阈值的交易、与新合约交互或执行其他敏感操作时，交易不会立刻执行，而是等待你的审批。

**你会在 Telegram 里收到通知。** OpenClaw 会在聊天中告诉你有一笔交易需要你确认，并附上审批链接。

**打开审批链接。** 点击 OpenClaw 在 Telegram 里发给你的链接，会直接跳转到该笔交易的审批页面。你也可以登录 https://wallet.example.com ，在 **Approvals**（审批）标签页查看所有待处理的审批请求。

**审批页面会显示交易详情：** 操作类型、金额、代币种类、发送方钱包、接收方地址、请求时间和剩余有效时间。仔细检查这些信息，确认是你授权 OpenClaw 执行的操作。

**批准或拒绝：**
- 点击 **Approve**（批准）并用指纹或面容验证，交易会自动签名并广播到区块链
- 点击 **Reject**（拒绝）取消交易，OpenClaw 会收到拒绝通知

**审批有效期为 30 分钟。** 超时未处理的请求会自动过期。如果错过了审批窗口，告诉 OpenClaw 重新发起操作即可。

## Web 界面功能概览

Web 界面（https://wallet.example.com）是你管理钱包和监督 OpenClaw 的控制面板。以下是你可以在 Web 界面上做的事情：

- **查看钱包和余额** —— 所有钱包的地址、链、余额一目了然
- **审批或拒绝交易** —— 在 Approvals 标签页处理 OpenClaw 发起的待审批请求
- **设置审批策略** —— 在钱包详情的 Policy 标签页设置 USD 阈值和日限额
- **查看合约白名单** —— 在 Whitelist 标签页查看已授权的合约（添加通过 OpenClaw 发起提案）
- **删除白名单中的合约** —— 直接在 Web 界面移除不再需要的合约授权
- **生成和撤销 API 密钥** —— 在 Account 标签页管理 OpenClaw 的访问凭证
- **查看操作历史** —— 在 History 标签页查看完整的审计记录，包括每一笔转账、审批、策略变更

日常使用中，你主要通过 Telegram 和 OpenClaw 聊天来操作钱包。Web 界面更多用于审批交易、调整安全设置和查看历史记录。

## 安全性

**私钥分片存储在 TEE 硬件节点。** 你的私钥在创建时被分割成多个碎片，分散存储在多个安全硬件节点中。签名时节点通过密码学协议协作完成，完整私钥从未被还原，也从未离开过安全硬件。

**OpenClaw 永远看不到你的私钥。** OpenClaw 只是发起签名请求，实际签名由 TEE 安全节点完成。即使 OpenClaw 被攻破，攻击者也无法获取你的私钥。

**大额交易需要你的指纹或面容。** 超过阈值的交易必须经过你的 Passkey 验证，任何人都无法伪造你的生物识别信息。

**消费限额由你控制。** 你设定 USD 阈值和日限额，OpenClaw 只能在这个范围内自主行动。

**随时可以撤销 OpenClaw 的访问权限。** 在 Web 界面的 Account 标签页删除 API 密钥，OpenClaw 的所有钱包操作权限瞬间归零。

## 常见问题

### 怎么安装钱包技能？

在 Telegram 上打开 OpenClaw 的对话，发送：**"安装这个 skill: https://github.com/TEENet-io/teenet-wallet/blob/master/skill/tee-wallet/SKILL.md"**。OpenClaw 会引导你完成设置，你需要提供钱包地址和 API 密钥。详细步骤请参考本指南的"第三步：在 OpenClaw 上安装钱包技能"。

### OpenClaw 不需要我审批就能做什么？

在你设定的规则范围内，OpenClaw 可以自主完成：
- 发送低于 USD 阈值的转账
- 查询任意钱包的余额和交易状态
- 与白名单中已开启自动审批的合约交互
- 查看钱包地址和公钥信息

具体边界取决于你的审批策略和白名单设置。阈值越高，OpenClaw 的自主权越大；阈值越低，你需要审批的频率越高。

### 什么操作一定需要我审批？

以下操作无论如何都需要你用 Passkey 确认：
- 超过 USD 阈值的任何转账
- 添加新合约到白名单
- 某些高风险操作（如代币授权），无论金额大小
- 删除钱包
- 删除账户
- 生成或撤销 API 密钥

这些操作涉及资产安全或权限变更，系统设计上不允许任何人（包括 OpenClaw）绕过你的确认。

### 可以用多个 OpenClaw 机器人吗？

可以。为每个 OpenClaw 实例生成一个独立的 API 密钥即可。它们共享同样的审批策略和白名单。你可以在历史记录中区分不同实例的操作，也可以单独撤销某个实例的密钥。建议为每个实例使用不同的标签名（比如"openclaw-交易"和"openclaw-理财"），方便管理和追踪。

### OpenClaw 花超了怎么办？

多重机制保护你的资产：
- **USD 阈值** —— 超过阈值的交易自动暂停等待你审批，OpenClaw 无法绕过
- **日限额** —— 当天累计金额达到上限后，OpenClaw 无法再发起任何交易
- **白名单** —— OpenClaw 只能与你授权的合约交互，无法将资金发送到未知合约
- **撤销密钥** —— 发现异常时，立即在 Web 界面撤销 API 密钥，OpenClaw 的所有权限瞬间归零

### 怎么停止 OpenClaw 使用我的钱包？

登录 https://wallet.example.com ，切换到 **Account** 标签页，在 API 密钥列表中找到对应的密钥，点击 **Revoke**（撤销）。密钥立即失效，OpenClaw 将无法执行任何钱包操作。如果你不确定哪个密钥对应哪个 OpenClaw 实例，可以撤销所有密钥，然后为需要保留的实例重新生成。

### 什么是 Passkey？

Passkey（通行密钥）是一种取代传统密码的身份验证方式，利用你设备上的指纹传感器、面容识别或硬件安全密钥来验证身份。和密码不同，Passkey 不需要记忆、无法被钓鱼、无法被暴力破解。在 TEENet Wallet 中，Passkey 是你审批交易和确认敏感操作的唯一方式。

### 丢了设备怎么办？

恢复方式取决于你的 Passkey 同步设置：
- **Apple 设备** —— Passkey 通过 iCloud 钥匙串自动同步，在新设备上登录同一个 Apple ID 即可恢复
- **Android / Chrome** —— Passkey 通过 Google 密码管理器同步，登录同一个 Google 账户即可
- **硬件安全密钥** —— 请确保有备用密钥，唯一的安全密钥丢失将无法访问账户

建议在多个设备上设置 Passkey（例如手机和电脑），这样即使丢失一个设备也不会失去访问权限。

### 支持哪些区块链？

**正式网络（有真实价值）：**
- 以太坊主网（Ethereum）—— 使用 ETH
- Optimism 主网 —— 以太坊二层网络，更快更便宜
- Solana 主网 —— 使用 SOL

**测试网络（免费练习用）：**
- Sepolia、Holesky —— 以太坊测试网
- Base Sepolia —— 以太坊二层测试网
- BSC Testnet —— 币安智能链测试网
- Solana Devnet —— Solana 测试网

此外，你可以通过 Web 界面的"Add Custom Chain"功能添加任何 EVM 兼容链（如 Polygon、Arbitrum、Avalanche 等）。
