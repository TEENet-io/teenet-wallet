# 常见问题

---

## 连接 Agent

### 如何连接 AI Agent？

在 [TEENet Wallet][wallet-url] 的 **设置** 面板中生成一个 API 密钥，然后把这个密钥和账户页显示的钱包 API URL 一起提供给你的 Agent。Agent 会用它们代表你调用钱包的 REST API。

**OpenClaw 示例：** 打开 OpenClaw 对话，发送：

```
Install this skill: https://github.com/TEENet-io/teenet-wallet/blob/main/skill/teenet-wallet/SKILL.md
```

OpenClaw 会要求你输入 API 密钥（以 `ocw_` 开头）和钱包 API URL。输入后，skill 就安装完成并可以使用。

如果你使用其他 Agent 平台，请参考该平台关于 API 凭据配置的文档。

## Agent 权限

### 哪些操作不需要审批？

你的 Agent 在无需 Passkey 的情况下可以执行以下操作：

- 发送低于 USD 审批阈值的转账
- 查询钱包余额和地址
- 查看交易历史
- 读取合约数据，例如查余额或查价格，但不发送交易

其余操作都需要你的确认。

### 哪些操作一定需要审批？

你的 Passkey 适用于超过 USD 审批阈值的转账、所有智能合约交互，以及白名单更新、策略修改、删除钱包和 API 密钥管理等钱包安全或策略变更。

如果转账会超过你的日消费限额，系统会直接拒绝，直到 UTC 午夜重置。审批页面和处理流程见 [审批与 Web 界面](/zh/user-approvals.md)。

### 可以同时连接多个 Agent 吗？

可以。为每个 Agent 单独生成一个 API 密钥即可。它们共享你的钱包和审批策略，但彼此独立。如果某个 Agent 行为异常，你可以只撤销它对应的密钥，而不影响其他 Agent。

## 安全

### 私钥是怎么保护的？

你的私钥会通过阈值密码学拆分到多个 TEE（可信执行环境）硬件节点中。没有任何单个节点、服务器或个人持有完整私钥。只有多个安全硬件节点共同协作，才能完成签名。你的 Agent 永远看不到私钥，它只会向钱包服务发送交易请求，再由钱包服务协调 TEE 节点完成签名。私钥始终不会离开安全硬件。

### 如果 Agent 想超额花钱怎么办？

如果 Agent 发起的交易超过你的阈值，该交易会进入待审批状态，在你批准之前不会发到链上。如果交易会超过你的日限额，则会被直接阻止，直到 UTC 午夜重置。日限额是基础设施层强制执行的硬限制。

### 我能看到 Agent 做过什么吗？

可以。Agent 的每一个动作都会记录在 [TEENet Wallet][wallet-url] Web 界面的 **活动** 标签页中。你可以随时查看，确认 Agent 的行为是否符合预期。

### 如何撤销 Agent 的访问权限？

进入 [TEENet Wallet][wallet-url]，打开 **设置**，找到该 Agent 正在使用的 API 密钥，然后点击 **Revoke**。使用 Passkey 完成验证后，该 Agent 会立即失去对你钱包的所有访问权限。如果以后还要重新连接，需要重新生成一个新的 API 密钥。

## 账户

### 什么是 Passkey？

Passkey 是密码的现代替代方案。你不再输入密码，而是通过指纹、Face ID 或硬件安全密钥来证明身份。你的生物识别数据不会离开设备，钱包只会收到你已成功完成认证的密码学证明。Passkey 不容易被钓鱼、猜测，也不会在数据泄露中像密码那样被直接盗用。你可以把它理解成“用解锁手机的方式来解锁钱包”。

### 丢了设备怎么办？

你的钱包和资金仍然安全，它们保留在 TEE 基础设施中，不会因为设备丢失而消失。你需要做的是重新获得账户访问权限。

很多 Passkey 系统，例如 Apple 的 iCloud Keychain 或 Google Password Manager，会在设备之间同步 Passkey。如果你丢了手机，但还有一台登录相同账户的笔记本电脑，通常仍然可以登录。有些平台也支持在新设备上通过扫描二维码，借助附近已登录设备完成认证。

如果你已经失去所有保存 Passkey 的设备访问权限，请联系钱包管理员了解账户恢复方案。

**最佳实践：** 确保你的 Passkey 已通过设备生态做备份，例如 Apple、Google，或安全保管的硬件安全密钥。

## 链支持

### 支持哪些区块链？

TEENet Wallet 支持区块链系统中主流的签名方案。标记 **✓** 的链已完成端到端测试。

| 方案 | 区块链 |
|--------|-------------|
| ECDSA / secp256k1 | Ethereum **✓**、Optimism **✓**、Base **✓**、BNB Chain **✓**、Avalanche **✓**、Arbitrum、Polygon、Bitcoin，以及任意 EVM 链 |
| EdDSA / ed25519 | Solana **✓** |

任何 EVM 兼容链都可以在运行时添加。可以让 Agent 列出可用链，或者直接查看 API 中的 `GET /api/chains`。
